You are building MahairaSystem — a unified web application for

================================================================
  SUPABASE DATABASE SCHEMA — CREATE THESE TABLES EXACTLY
================================================================

Run this SQL in the Supabase SQL editor:

-- Products table
CREATE TABLE products (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name        TEXT NOT NULL,
  category    TEXT NOT NULL CHECK (category IN ('Paints','Iron & Steel','Hardware','Tools')),
  description TEXT,
  cost_price  NUMERIC(10,2) NOT NULL,
  selling_price NUMERIC(10,2) NOT NULL,
  stock_qty   INTEGER NOT NULL DEFAULT 0,
  reorder_level INTEGER NOT NULL DEFAULT 5,
  image_url   TEXT,
  is_active   BOOLEAN NOT NULL DEFAULT true,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- Customers table
CREATE TABLE customers (
  id         UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name       TEXT NOT NULL,
  phone      TEXT NOT NULL,
  address    TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Bills table
CREATE TABLE bills (
  id             UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  customer_id    UUID REFERENCES customers(id),
  subtotal       NUMERIC(10,2) NOT NULL,
  gst_rate       NUMERIC(5,2) NOT NULL DEFAULT 18,
  gst_amount     NUMERIC(10,2) NOT NULL,
  total_amount   NUMERIC(10,2) NOT NULL,
  payment_status TEXT NOT NULL DEFAULT 'Outstanding'
                 CHECK (payment_status IN ('Paid','Outstanding','Overdue')),
  notes          TEXT,
  created_at     TIMESTAMPTZ DEFAULT now()
);

-- Bill items table (line items per bill)
CREATE TABLE bill_items (
  id           UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  bill_id      UUID REFERENCES bills(id) ON DELETE CASCADE,
  product_id   UUID REFERENCES products(id),
  product_name TEXT NOT NULL,   -- snapshot: store name at time of billing
  unit_price   NUMERIC(10,2) NOT NULL, -- snapshot: price at time of billing
  quantity     INTEGER NOT NULL,
  line_total   NUMERIC(10,2) NOT NULL
);

-- Payments table
CREATE TABLE payments (
  id         UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  bill_id    UUID REFERENCES bills(id) ON DELETE CASCADE,
  amount_paid NUMERIC(10,2) NOT NULL,
  paid_at    TIMESTAMPTZ DEFAULT now(),
  notes      TEXT
);


================================================================
  SUPABASE ROW LEVEL SECURITY (RLS) — APPLY EXACTLY AS BELOW
================================================================

Run this SQL in the Supabase SQL editor after creating tables:

-- Enable RLS on all tables
ALTER TABLE products  ENABLE ROW LEVEL SECURITY;
ALTER TABLE customers ENABLE ROW LEVEL SECURITY;
ALTER TABLE bills     ENABLE ROW LEVEL SECURITY;
ALTER TABLE bill_items ENABLE ROW LEVEL SECURITY;
ALTER TABLE payments  ENABLE ROW LEVEL SECURITY;

-- Products: anyone can READ (public catalogue)
CREATE POLICY "public can read active products"
  ON products FOR SELECT
  USING (is_active = true);

-- Products: only authenticated owner can write
CREATE POLICY "owner can manage products"
  ON products FOR ALL
  USING (auth.uid() IS NOT NULL)
  WITH CHECK (auth.uid() IS NOT NULL);

-- All other tables: owner only
CREATE POLICY "owner only" ON customers FOR ALL USING (auth.uid() IS NOT NULL);
CREATE POLICY "owner only" ON bills     FOR ALL USING (auth.uid() IS NOT NULL);
CREATE POLICY "owner only" ON bill_items FOR ALL USING (auth.uid() IS NOT NULL);
CREATE POLICY "owner only" ON payments  FOR ALL USING (auth.uid() IS NOT NULL);

-- Supabase Storage: create bucket "product-images" as PUBLIC
-- Images are public-read, authenticated-write (set in Supabase dashboard)


================================================================
  AUTHENTICATION — SUPABASE AUTH
================================================================

Method: Email + password (supabase.auth.signInWithPassword)

IMPORTANT: There is NO signup page in the app. The owner account
is created manually in the Supabase Dashboard under Authentication
→ Users → Add User. The app only has a login page.

AuthContext (src/context/AuthContext.jsx):
- On mount, call supabase.auth.getSession() to restore session
- Listen to supabase.auth.onAuthStateChange for live updates
- Expose: { session, user, loading, signOut }

ProtectedRoute component:
- Wraps all /owner/* routes
- If loading: show spinner
- If no session: redirect to /owner/login
- If session exists: render children

Login page (/owner/login):
- Uses supabase.auth.signInWithPassword({ email, password })
- On success: navigate to /owner/dashboard
- On error: show "Invalid email or password" inline
- No signup link, no forgot password visible

================================================================
  OWNER DASHBOARD — PAGE REQUIREMENTS
================================================================

DASHBOARD (/owner/dashboard)
KPI cards (fetch from Supabase):
- Today's Sales: SUM of bills.total_amount WHERE DATE(created_at) = today AND payment_status = 'Paid'
- Monthly Sales: SUM for current month
- Outstanding Dues: SUM WHERE payment_status = 'Outstanding'
- Low Stock Count: COUNT of products WHERE stock_qty <= reorder_level AND is_active = true
Alert banners below KPIs:
- If low stock count > 0: show warning alert with link to /owner/inventory
- If overdue bills exist: show danger alert with link to /owner/bills

INVENTORY (/owner/inventory)
- Data table of all products (including inactive)
- Columns: Image, Name, Category, Selling Price, Stock, Status, Actions
- Add Product button → opens product form in slide-over/modal
- Edit row → opens pre-filled product form
- Delete row → soft delete: SET is_active = false (NOT hard delete)
  Show confirmation dialog before deleting
- Stock badge: green if ok, orange if low, red if 0
- Product image upload: upload to Supabase Storage "product-images" bucket,
  store returned public URL in products.image_url

CUSTOMERS (/owner/customers)
- Data table: Name, Phone, Address, Total Bills, Date Added, Actions
- Add Customer button → opens customer form
- Edit/Delete customer
- Click customer name → show their bill history in a modal

BILLS (/owner/bills)
- Data table: Bill #, Customer, Date, Total, Status, Actions
- Status badges: Paid (green), Outstanding (amber), Overdue (red)
- Mark as Paid button per row → updates payment_status to 'Paid',
  creates a record in payments table
- View Bill button → opens bill detail modal with print option
- New Bill button → navigates to /owner/bills/new
- Auto-mark bills as Overdue if payment_status = 'Outstanding' AND
  created_at < now() - interval '30 days'

BILL GENERATION (/owner/bills/new)
Step 1 — Select customer (searchable dropdown from customers table)
Step 2 — Add line items:
  - Product selector (searchable, shows name + stock + price)
  - Quantity input (validate: cannot exceed stock_qty)
  - Unit price auto-filled from products.selling_price
  - Line total = qty × unit_price (auto-calculated)
  - Add/remove rows
Step 3 — Totals (auto-calculated, shown live):
  - Subtotal = sum of all line totals
  - GST (18% default, editable) = subtotal × gst_rate / 100
  - Grand Total = subtotal + gst_amount
Step 4 — Payment status: Paid or Outstanding
Step 5 — "Generate Bill" button:
  - INSERT into bills table
  - INSERT all rows into bill_items table
  - UPDATE products.stock_qty -= quantity for each item
  - Show success toast
  - Open printable bill view / generate PDF using jsPDF

BILL PDF format (jsPDF):
  Header: Store name, address, phone, GST number (placeholder)
  Bill details: Bill number, date, customer name, phone, address
  Items table: #, Product, Qty, Unit Price, Total
  Totals: Subtotal, GST (rate%), Grand Total
  Footer: "Thank you for your business"

REPORTS (/owner/reports)
Tab 1 — Daily (last 7 days):
  Bar chart: X axis = date, Y axis = total sales in ₹
  Data: GROUP BY DATE(created_at) on bills WHERE payment_status = 'Paid'

Tab 2 — Monthly (last 6 months):
  Bar chart: X axis = month, Y axis = total sales in ₹

Tab 3 — Top Products:
  Table: Product name, Units sold, Revenue generated
  Data: JOIN bill_items → GROUP BY product_name → ORDER BY qty DESC

All charts use Recharts. All data fetched from Supabase.


================================================================
  PRODUCT IMAGE HANDLING
================================================================

Owner adds a product:
1. Owner picks an image file in the product form
2. Upload to Supabase Storage: supabase.storage
   .from('product-images')
   .upload(`${productId}-${Date.now()}`, file)
3. Get public URL: supabase.storage
   .from('product-images')
   .getPublicUrl(filePath)
4. Save the public URL to products.image_url

Public catalogue:
- Render <img src={product.image_url} /> if image_url is not null
- Fallback: show a grey placeholder with the product initial if no image

Deleting a product (soft delete):
- Set is_active = false in the database
- Do NOT delete the image from storage (bill history may reference it)

  CURRENCY AND NUMBER FORMATTING

      maximumFractionDigits: 2
